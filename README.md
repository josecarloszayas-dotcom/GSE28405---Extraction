Analyzing GSE28405 presented several challenges. First, the dataset lacked gender information entirely, requiring computational sex prediction. Second, massiR's built-in Y-probe lists didn't match the dataset's GI identifiers, necessitating a custom probe list created via biomaRt and GPL annotation. Third, initial massiR runs on timepoint1 patients alone gave only 9 males versus the paper's reported 16. Fourth, running massiR on each timepoint separately produced wildly inconsistent results (2, 29, and 1 male), suggesting Y chromosome expression fluctuates during dengue infection. Fifth, controls were spread across all timepoint files, requiring careful extraction. Sixth, patient matching across timepoints required manual mapping since the same individual has different GSM IDs at each timepoint. Seventh, I discovered that pooling patients and controls confounded clustering, so I analyzed them separately. Eighth, creating final files required combining patient data from each timepoint with control data from timepoint1 without duplication. Ninth, I had to ensure sex assignments remained consistent for the same patient across all three timepoints. Tenth, the final files needed identical sample ordering between expression and sex CSVs for downstream MCO analysis.
